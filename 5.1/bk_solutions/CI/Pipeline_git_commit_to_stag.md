# 测试环境自动更新

## 情景
开发同学完成功能开发、本地测试通过后，运维通过自动化工具或人工登录服务器更新，由于开发和运维存在 **沟通成本**，导致持续集成的测试环节效率低下，影响整体研发效率。

接下来看蓝盾（BK-CI）是如何做到 **测试环境自动更新**。

## 前提条件

- [部署蓝盾（BK-CI）](https://github.com/Tencent/bk-ci)

## 操作步骤
- 配置自动构建流水线
- 提交代码（Git Commit）验证
